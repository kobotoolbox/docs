# Validación de registros

El propietario de un proyecto puede otorgar el permiso "Puede Validar" a otros usuarios. Los usuarios con este permiso pueden ver un registro, editarlo si es necesario, y asignarle un estado al registro en cuestión. Asignar un estado a un registro/envío en particular eleva los estándares de recolección de datos para equipos con más de un encuestador.

## ¿Cuáles son los estados que puedes asignar a un registro?

Las etiquetas de estado de validación disponibles incluyen:

* **En espera**: El registro está en revisión.
* **Aprobado**: Los datos de este registro son precisos.
* **Marcado para eliminación**: Los datos de este registro deben eliminarse del conjunto de datos.

## Descripción general de la funcionalidad

* Este nuevo permiso añade una columna "Validado" a la tabla de datos de tu proyecto.

    ![image](/images/record_validation/validated.png)

* Los usuarios pueden filtrar la vista de tabla según el estado de validación o cualquier otra pregunta de selección única. Por ejemplo, puedes usar estos filtros para ver los envíos asociados a un encuestador en particular, en el Distrito X. Es posible usar más de un filtro a la vez.

    ![image](/images/record_validation/filter.png)

* Establece un estado de validación para varios registros, o para todos, a la vez.

    Para actualizar varios registros, selecciónalos haciendo clic en la casilla de verificación a la izquierda del registro y luego haz clic en "Actualizar seleccionados" para seleccionar el nuevo estado.

    Para actualizar todos los registros de la página, selecciona la casilla de verificación en la parte superior izquierda; o para actualizar todos los registros de tu conjunto de datos, selecciona el texto en azul que dice "Seleccionar todos los XXX". Luego, haz clic en "Actualizar seleccionados" para seleccionar el nuevo estado.

    ![image](/images/record_validation/select.png)
